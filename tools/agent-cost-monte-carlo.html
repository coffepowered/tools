1. Simulation Objective

This tool estimates the probability distribution of hourly API cost ($/hour) for a single, serial AI agent running continuously. It uses the Monte Carlo method to propagate uncertainty across all operational parameters simultaneously: latency variability, payload size, and cache efficiency.

Each scenario is an independent random draw. After n draws, the aggregated distribution reveals expected costs, budget targets, and tail risks that deterministic estimates cannot capture.

3. Calculation Logic

Sampling

Most parameters are drawn from a uniform (linear) distribution over their configured range:

$$X_i \sim \mathcal{U}[\text{Min},\ \text{Max}]$$

Context Input tokens are an exception: they are sampled from a log-uniform distribution, giving equal probability mass to each order of magnitude across the range. This prevents the 100K end from dominating the distribution when the range spans 1K–100K:

$$X_i = \exp\!\left(\mathcal{U}[\ln(\text{Min}),\ \ln(\text{Max})]\right)$$

Cost per API Call

$$T_{\text{cached}} = T_{\text{context}} \times r_{\text{cache}}$$ $$T_{\text{fresh}} = T_{\text{context}} - T_{\text{cached}}$$ $$C_{\text{call}} = \frac{T_{\text{fresh}} \cdot P_{\text{in}} + T_{\text{cached}} \cdot P_{\text{cache}} + T_{\text{out}} \cdot P_{\text{out}}}{10^6}$$

Calls per Hour

$$\text{Calls/hr} = \frac{3600}{L} \times d$$

where $L$ is sampled latency (s) and $d$ is the sampled duty cycle (fraction 0–1).

Hourly Cost

$$C_{\text{hour}} = C_{\text{call}} \times \text{Calls/hr}$$

Statistics

$$\bar{C} = \frac{1}{n}\sum_i C^{(i)}_{\text{hour}}, \quad P_{k} = C^{(\lfloor k \cdot n\rfloor)}_{\text{hour}}$$

The histogram y-axis shows percentage of scenarios falling in each bin, so that run count does not affect visual scale.

5. Assumptions & Limitations

No Parallelism

The model simulates exactly one serial agent: one call at a time, no concurrent execution. Worker pools, fan-out graphs, and multi-step pipelines require a separate model.

Thinking Tokens Not Separated

Models with extended reasoning (Claude Opus, GPT-5.4 reasoning mode) bill thinking tokens at output prices. They are folded into the Output parameter. For reasoning-heavy workloads the actual cost may be higher.

Parameter Independence

Monte Carlo sampling assumes all parameters are independent. In practice, latency and output-token count are positively correlated, as are cache hit ratio and context size. This can understate variance at the tails.

Fixed Pricing

Pricing is static for the session. Volume discounts, Batch API pricing (−50%), time-of-day variability, and cache write costs are not modelled.

Partial Logarithmic Sampling

Context Input tokens use log-uniform sampling, correctly weighting small and large contexts equally. All other parameters (latency, output tokens, duty cycle, cache ratio) still use linear (uniform) sampling. Future versions may extend log-sampling to output tokens as well.

Empirical Latency Thresholds

The task-type boundaries in §4 are based on field observations, not official provider benchmarks. They should be treated as indicative.
